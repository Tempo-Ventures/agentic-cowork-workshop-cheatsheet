Bezpečné používání

Permission modes

plan Jen čte a navrhuje plán. Nic nezmění, dokud neschválíte. Průzkum claude --permission-mode plan
default Ptá se před každou změnou i příkazem. Výchozí claude
acceptEdits Soubory upravuje sám, příkazy schvalujete. Úpravy souborů claude --permission-mode acceptEdits
auto Schvaluje i příkazy, pokud je systém Anthropicu vyhodnotí jako bezpečné. Sám blokuje odesílání dat ven, nasazení do produkce a nebezpečné příkazy. Vyžaduje plán Max, Team, Enterprise nebo API. Delší úkoly claude --permission-mode auto
bypassPermissions Bez kontrol — schvaluje vše kromě chráněných cest (viz níže). Používejte jen v kontejneru nebo virtuálním stroji. Bez kontrol claude --dangerously-skip-permissions

Shift+Tab přepíná mezi režimy za běhu (aktuální vidíte ve spodní liště). /permissions zobrazí a upraví pravidla (co je povolené, co se ptá, co je zakázané).

Co agentovi nedávat

  • Hesla, přístupové klíče, čísla karet
  • Citlivé údaje klientů (rodná čísla, zdravotní data, smlouvy bez mandátu)
  • Cizí soubory, ke kterým nemáte oprávnění
Únik
Heslo nebo klíč berte jako kompromitované -- zneplatněte je a vygenerujte nové. Data, která agent viděl, jsou už na serverech Anthropicu; /clear je neodstraní, jen začne novou konverzaci. Citlivý soubor smažte z pracovní složky i z gitu (git rm + nový commit).

Oddělená pracovní složka

  • Agenta spouštějte v nové, prázdné složce — ne na ploše, v Dokumentech ani v domově
  • Agent vidí jen tuto složku a její podsložky
  • Další složku přidejte vědomě: /add-dir cesta

Sandbox

Omezí, k jakým souborům a serverům může agent při spouštění příkazů přistupovat — ochrana vynucená operačním systémem, ne agentem. Zapíná se v souboru settings.json.

"sandbox": { "enabled": true }
Zapnutí v settings.json
macOS
Funguje (přes Seatbelt)
Linux
Funguje (vyžaduje novější jádro)
Windows
Nativně není — jen ve WSL2

Chráněné cesty

Vždy se ptá (i v bypassPermissions): .git, .claude (kromě commands, agents, skills), .vscode, .idea, .husky. V default také shell config (.bashrc, .zshrc, .profile) a globální git (.gitconfig, .gitmodules).

Pravidlo
Co schválíte, to agent udělá. V režimu default odsouhlasíte každou akci předem. Volnější režimy (acceptEdits, auto, bypassPermissions) pouštějte jen v izolované složce a s aktivním gitem (viz výše).
