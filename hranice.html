/System · /etc · jiní uživatelé · síťové disky · cizí domovské složky
Mimo pracovní složku 👁 Ptá se na čtení i zápis
~/Documents · ~/Desktop · rodičovská složka · jiné projekty. V bypassu se neptá.
Pracovní složka + podsložky ✓ Ptá se před každou změnou
složka, kde jste spustili claude, plus všechny její podsložky. Volnější módy (acceptEdits, auto, bypass) ptaní zkracují — viz tabulka.
.git · .claude · .vscode · .idea · .husky 🔒 Vždy se ptá (i v bypassu)
v default také .bashrc, .zshrc, .gitconfig

claude --add-dir /jina/cesta nebo /add-dir cesta rozšíří „Pracovní složku" o další konkrétní složku. Bez toho ji agent nevidí, i kdybyste mu cestu napsali do promptu.

Jak to číst

Diagram a labely platí pro default mód. Volnější módy mění, jak často se Claude ptá uvnitř pracovní zóny. Bypass jediný přeskakuje ptaní i mimo ni — chráněné cesty (.git, .husky…) se ale ptá vždy.

Vlastní hranici (kam Claude vůbec vidí) nastavujete vy: vyberete pracovní složku, případně přidáte další přes --add-dir.

Sandbox navíc
Volitelné druhé patro: zapnete "sandbox": { "enabled": true } v settings.json a operační systém dál zužuje hranice (Claude pak nemůže ani číst mimo zónu, ani sahat na .bashrc v bypass módu). macOS přes Seatbelt, Linux/WSL2 přes bubblewrap, na Windows nativně zatím není.
